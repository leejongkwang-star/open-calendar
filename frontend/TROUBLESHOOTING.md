# 문제 해결 가이드

## Network Error 발생 시

### 원인
프론트엔드가 백엔드 API 서버에 연결하지 못할 때 발생합니다.

### 해결 방법

#### 1. 백엔드 서버 실행 확인

**새 터미널 창을 열고:**
```bash
cd 캘린더백엔드
npm run dev
```

백엔드 서버가 정상적으로 실행되면:
```
🚀 서버가 http://localhost:3001에서 실행 중입니다.
```

#### 2. 프론트엔드 서버 실행 확인

**다른 터미널 창에서:**
```bash
cd 캘린더프론트엔드
npm run dev
```

프론트엔드 서버가 정상적으로 실행되면:
```
VITE v5.0.8  ready in xxx ms
➜  Local:   http://localhost:5173/
```

#### 3. 환경 변수 확인

`캘린더프론트엔드/.env` 파일이 있고 다음 내용이 포함되어 있는지 확인:
```
VITE_API_BASE_URL=http://localhost:3001/api
```

#### 4. 서버 상태 확인

브라우저에서 직접 확인:
- 백엔드: http://localhost:3001/health
- 프론트엔드: http://localhost:5173

### 임시 해결책 (Mock 모드 사용)

백엔드 서버를 실행할 수 없는 경우, Mock 모드를 사용할 수 있습니다:

`.env` 파일을 수정하거나 삭제:
```env
# 주석 처리하거나 삭제
# VITE_API_BASE_URL=http://localhost:3001/api

# 또는 Mock 모드 명시
VITE_USE_MOCK=true
```

그 후 프론트엔드 서버를 재시작하세요.

### 일반적인 문제

1. **포트 충돌**
   - 백엔드: 포트 3001이 이미 사용 중
   - 프론트엔드: 포트 5173이 이미 사용 중
   - 해결: 다른 포트로 변경하거나 사용 중인 프로세스 종료

2. **CORS 오류**
   - 백엔드의 CORS 설정 확인
   - `캘린더백엔드/src/server.js`에서 CORS_ORIGIN 확인

3. **환경 변수 미적용**
   - `.env` 파일 수정 후 프론트엔드 서버 재시작 필요
